# Support

How to get help with Gaugid SDK.

## Community support

- **Questions and ideas** — [GitHub Discussions](https://github.com/alpibrusl/gaugid-sdk/discussions)  
  Use for “how do I…?”, design ideas, and general questions. Check existing threads before posting.

- **Bugs and feature requests** — [GitHub Issues](https://github.com/alpibrusl/gaugid-sdk/issues)  
  Use the issue templates for bugs and features. Include version, OS, and a minimal example where relevant.

- **Documentation** — [docs.gaugid.com](https://docs.gaugid.com)  
  Start here for installation, quick start, and API usage.

## Security

**Do not** report security vulnerabilities in public issues or discussions.  
See [SECURITY.md](SECURITY.md) for how to report them privately.

## Code of conduct

Our [Code of Conduct](CODE_OF_CONDUCT.md) applies to all community spaces. Report conduct concerns as described there.

## Commercial and enterprise support

For commercial support, SLAs, or enterprise licensing, contact Gaugid via [gaugid.com](https://gaugid.com).

## Response expectations

- **Issues and discussions** — Maintainers aim to respond within a few business days. Community members may also help.
- **Pull requests** — We review PRs as capacity allows; see [CONTRIBUTING.md](CONTRIBUTING.md) for how to contribute.

Thank you for using Gaugid SDK.
