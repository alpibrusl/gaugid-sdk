# Protocol Extension Proposals

This directory contains proposals for extending the a2p protocol specification.

## Current Proposals

### [Memory Types Extension](./memory-types-protocol-extension.md)

**Status**: Draft for Review  
**Proposed**: 2026-01-15  
**Target Version**: 1.1.0

Proposes adding semantic and procedural memory types to the a2p protocol specification to align with modern agent framework architectures (LangGraph, Google ADK).

## Proposal Process

1. **Draft**: Create proposal document in this directory
2. **Review**: Submit to a2p protocol team for review
3. **Discussion**: Gather feedback from community
4. **Approval**: Protocol team approves or requests changes
5. **Implementation**: If approved, implement in protocol spec and SDKs

## Proposal Template

When creating a new proposal, use this structure:

```markdown
# Proposal Title

**Proposal Date**: YYYY-MM-DD
**Proposed By**: [Team/Individual]
**Status**: Draft/Under Review/Approved/Rejected
**Target Protocol Version**: X.Y.Z

## Executive Summary
[Brief overview]

## Motivation
[Why this change is needed]

## Proposed Changes
[Specific changes to protocol]

## Use Cases
[Examples of how it would be used]

## Backward Compatibility
[How existing implementations are affected]

## Implementation Considerations
[Technical details]

## Timeline
[Estimated implementation timeline]

## Open Questions
[Questions for discussion]

## Approval
[Decision tracking]
```

## Contributing

To propose a protocol extension:

1. Create a new markdown file in this directory
2. Follow the proposal template
3. Submit for review via:
   - a2p protocol team meeting
   - GitHub issue/discussion
   - Direct communication with protocol maintainers
